# Checklist — 核心阅读体验

## 数据解耦

| # | 验收项 | 通过条件 | 代码证据 |
|---|-------|---------|---------|
| 1.1 | INDEX.json 格式正确 | 文件存在，包含所有故事的 id/title/category/wordCount/createdAt | `stories/INDEX.json` |
| 1.2 | 每篇故事独立 JSON 文件 | 文件存在，字段包含完整 content | `stories/001-纸人.json` |
| 1.3 | 字数准确 | wordCount 与 content 实际字符数匹配 | 检查 wordCount 值 |
| 1.4 | STORIES 数组已移除 | `reader.html` 无 `const STORIES` 声明 | grep "STORIES" |
| 1.5 | 数据从外部 JSON 加载 | 页面加载后 network 面板可见 INDEX.json 请求 | DevTools Network |

## 随机推荐

| # | 验收项 | 通过条件 | 代码证据 |
|---|-------|---------|---------|
| 2.1 | 打开页面直接进入随机故事 | 页面加载后显示的是故事正文，非列表页 | 手动测试 |
| 2.2 | "换一篇"按钮存在 | 页面底部有"换一篇"按钮，点击后切换为另一篇随机故事 | 查看底部导航栏 |
| 2.3 | 随机算法避免连续同题材 | 连续点击"换一篇"10 次，相邻两篇题材不同 | 手动测试 |
| 2.4 | 全题材标签栏可见 | 阅读器顶部可见 8 个题材标签横排 | 手动查看 |
| 2.5 | 标签筛选生效 | 点击"赶尸"标签后，仅显示赶尸题材故事 | 手动测试 |
| 2.6 | 再次点击标签取消筛选 | 已选中标签再次点击后取消高亮，恢复全随机 | 手动测试 |
