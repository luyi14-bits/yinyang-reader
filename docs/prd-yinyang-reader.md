## 1. Executive Summary

一款 **PWA 极简阅读器**，每次打开随机推送一篇阴阳先生/道教志怪类高质量短篇小说（500-1000 字），面向办公室摸鱼场景，内容由写手 Agent 生产并经人工把关，不做长篇、不做 AI 味。

---

## 2. Problem Statement

| 用户痛点 | 业务影响 |
|---------|---------|
| 想看阴阳先生题材短篇但懒得搜、不好找 | 市场缺少专注于该垂类的极简阅读产品 |
| 网文质量参差，AI 生成味重，读不下去 | 内容质量是核心竞争壁垒 |
| 长篇连载门槛高，摸鱼 5 分钟读不完 | 产品形态必须适配碎片化场景 |
| 现有产品（起点/每天读点故事）体验太重 | 用户需要的是"随手打开就能刷一篇" |

**一句话**：为想摸鱼 5 分钟读一篇高质量民俗志怪短篇的上班族，提供一个零摩擦的随机阅读器。

---

## 3. Goals & Metrics

### 3.1 SMART 目标（V1）

| 目标 | 指标 | 衡量方式 |
|------|------|---------|
| 用户能在 3 秒内开始阅读 | 页面加载 → 首屏文字可见 < 3s | Lighthouse / 实际测试 |
| 单篇阅读完成率 > 60% | 滚动到底部的事件触发比例 | 前端埋点 |
| 用户每天至少打开一次 | DAU/MAU 比率（暂不设硬指标，V2 再追踪） | — |
| 8 个题材全部可筛选 | 题材标签全部可点击过滤 | 功能验收 |

### 3.2 V1 不做（不追踪）的指标

- 用户注册/登录数据（纯免费，无需账号）
- 收入/广告数据（纯免费，无商业化）
- 分享/传播数据（V1 无社交功能）
- 内容库存数量（写手 Agent 负责，PM 不卡库存门槛）

## 5. User Stories

### 5.1 核心用户故事（Gherkin 格式）

```gherkin
Feature: 随机阅读阴阳先生短篇

  Scenario: 打开网站即可阅读一篇随机故事
    Given 用户首次打开 PWA
    When 页面加载完成
    Then 直接展示一篇完整的短篇故事
    And 故事字数在 500-1000 字之间
    And 故事内容符合"阴阳先生志怪"题材

  Scenario: 看完一篇后换一篇新的
    Given 用户已阅读完当前故事
    When 用户点击"换一篇"按钮
    Then 随机展示另一篇未读过的故事
    And 避免连续出现同一题材

  Scenario: 按题材筛选想看的故事
    Given 用户在主界面
    When 用户点击题材标签（如"赶尸"）
    Then 随机展示一篇该题材的故事
    And 标签高亮显示当前筛选状态

  Scenario: 清除题材筛选恢复全随机
    Given 用户当前按"赶尸"筛选
    When 用户再次点击已选中的标签（或点击"全部"）
    Then 清除筛选，恢复全随机模式

  Scenario: 摸鱼时快速伪装为Outlook
    Given 用户正在阅读故事
    When 用户按 Esc 键或双击空白区域
    Then 页面替换为 Outlook 邮件列表伪装界面
    And 伪装界面包含左侧导航栏、中间邮件列表、右侧预览窗格
    And 邮件列表含真实感标题/发件人/时间
    And 再次触发恢复故事阅读

  Scenario: PWA 离线使用
    Given 用户已访问过网站
    When 用户处于离线状态打开 PWA
    Then 仍能阅读已缓存的故事
    And 离线故事不少于最近 5 篇

  Scenario: 添加到主屏幕
    Given 用户在浏览器中打开网站
    When 浏览器提示"添加到主屏幕"
    Then 用户确认后可像原生 App 一样从桌面图标打开
```

### 5.2 V1 优先级（MoSCoW）

| 优先级 | 故事 |
|--------|------|
| **Must** | 打开即读、换一篇、题材筛选 |
| **Should** | PWA 离线缓存、添加到主屏幕、Outlook 伪装模式 |
| **Could** | — |
| **Won't (V1)** | 用户账号、收藏、历史记录、阅读进度同步 |

## 6. Functional Requirements

### 6.1 核心阅读功能

| ID | 功能 | 详细说明 |
|----|------|---------|
| FR-01 | 故事展示 | 纯文字排版，标题 + 正文，无封面图、无作者头像 |
| FR-02 | 随机推荐 | 默认从全部故事池完全随机选取，避免连续两篇同题材 |
| FR-03 | 换一篇 | 按钮位于页面底部（读完自然到达），也可于顶部放置快捷入口 |
| FR-04 | 题材筛选 | 8 个题材标签横排展示，单选模式，再次点击取消筛选 |
| FR-05 | 加载状态 | 首屏加载 < 3s，切换故事过渡动画 < 300ms |

### 6.2 PWA 特性

| ID | 功能 | 详细说明 |
|----|------|---------|
| FR-06 | Service Worker | 缓存静态资源 + 最近 N 篇故事，支持离线阅读 |
| FR-07 | Web Manifest | 支持"添加到主屏幕"，独立窗口打开（无浏览器地址栏） |
| FR-08 | 响应式布局 | 适配手机竖屏（375-428px 宽为主），横屏和桌面端兜底 |

### 6.3 摸鱼模式（Should）

| ID | 功能 | 详细说明 |
|----|------|---------|
| FR-09 | 一键伪装 | 按 Esc 键或双击空白区域，切换为伪装界面；再次触发恢复故事阅读 |
| FR-10 | 伪装界面 | 伪装为 **Outlook 邮件列表界面**，包含：左侧导航栏（收件箱/已发送/草稿等）、中间邮件列表（含真实感邮件标题/发件人/时间）、右侧预览窗格。伪装界面需静态可交互（可滚动），不连接真实邮箱 |

### 6.4 内容数据格式

```typescript
interface Story {
  id: string;           // 唯一标识，如 "gan-shi-001"
  title: string;        // 故事标题，如 "桥头纸人"
  category: Category;   // 题材分类
  content: string;      // 正文（纯文本，含「」对话标记）
  wordCount: number;    // 字数
  createdAt: string;    // 创作日期 ISO 8601
}

type Category = 
  | '夜路遇祟' | '赶尸' | '风水局' | '算命摊'
  | '捉鬼驱邪' | '扎纸' | '收魂' | '过阴';
```

## 7. Non-Functional Requirements

| 类别 | 要求 | 衡量标准 |
|------|------|---------|
| 性能 | 首屏渲染 < 3s（3G 网络） | Lighthouse Performance ≥ 90 |
| 性能 | 故事切换 < 300ms | 前端计时 |
| 离线 | Service Worker 缓存最近 5 篇故事 | 离线可读 |
| 可访问性 | WCAG 2.1 AA 级 | 对比度 ≥ 4.5:1，支持系统字体缩放 |
| 安全性 | 纯静态站点，无用户输入，无后端 | XSS 风险极低 |
| 兼容性 | iOS Safari 15+ / Android Chrome 90+ | 主流移动浏览器 |
| PWA | Lighthouse PWA 评分 ≥ 90 | manifest + SW 完整 |
| 代码质量 | 纯前端项目，零后端依赖 | 单次 `npm install && npm build` 可部署 |

## 8. Technical Considerations

### 8.1 架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 架构模式 | 纯静态站点（SSG） | 无用户系统、无实时数据，静态站点最简 |
| 前端框架 | React / Vue / 原生（待 Spec 阶段评估） | 页面交互简单，轻量框架即可 |
| 内容存储 | 静态 JSON/Markdown 文件，构建时打包 | 无需 CMS，内容更新 = 提交新文件 + 重新构建 |
| 部署 | 静态托管（GitHub Pages / Vercel / Netlify） | 免费、零运维 |
| PWA | Workbox / vite-plugin-pwa | 标准方案，文档齐全 |

### 8.2 内容流水线

```
写手 Agent 生成故事
    → 人工审稿（AI味检查 + 民俗准确性）
    → 存入 stories/ 目录（Markdown 格式）
    → Git 提交触发 CI
    → 自动构建 + 部署
```

### 8.3 数据流

```
用户打开 PWA
    → Service Worker 拦截请求
    → 优先返回缓存（离线可用）
    → 在线时更新缓存
    → 前端从静态 JSON 读取故事池
    → 随机算法选取一篇
    → 渲染展示
```

## 10. Out of Scope

| 不做（V1） | 原因 | 何时考虑 |
|------------|------|---------|
| 用户注册/登录 | 纯免费，无需账号体系 | V2 |
| 收藏/历史记录 | 增加复杂度，V1 验证核心体验 | V2 |
| 评论/点赞/分享 | 不做社交 | V2+ |
| 长篇连载 | 场景不匹配 | 永不 |
| AI 直接生成发布 | 老板明确"不要 AI 的" | 永不 |
| 听书/语音朗读 | 非核心 | V3+ |
| 夜间模式/字体调节 | V1 基础默认即可 | V2 |
| 多语言 | 内容为民俗中文，翻译失真 | V3+ |
| 商业化（广告/订阅） | 先验证留存 | V2+ |
| iOS/Android 原生 App | PWA 先行验证 | V2+ |
| 内容管理系统 (CMS) | 静态文件够用 | 内容量 > 100 篇时评估 |
