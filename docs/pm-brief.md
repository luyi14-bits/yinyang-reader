## 一、需求来源

**老板原话整理：**

> "那种玄幻小故事"  "就是一个几百字的短篇小说也可以"  "就是随机给个玄幻故事看看"  "那种道教、阴阳先生那种"  "不要AI的，写的不好看"  "懒得搜这种阴阳先生小说"  "就跟你这算卦的一样"

**核心诉求概括：**
- 内容是阴阳先生/道教志怪类短篇小说
- 单篇 500-1000 字，5 分钟内读完
- 随机推荐，不用自己翻找
- 写作质量要高，不像 AI 写的
- 使用场景：上班摸鱼，手机上快速浏览

### 3.2 经典桥段模板

每个题材都有固定的"起承转合"套路，实际是民间口述传统的叙事基因：

```
【起】师父说过一个规矩 → 偏偏破了 → 出事了
【承】徒弟（我）独自面对 → 看见/听见不对劲的细节
【转】师父出手（或者徒弟自己悟了） → 真相半露
【合】留一个不答完的问题 → 结尾一句话收住
```

## 四、写作风格规范

### 4.1 语言要求

| ✅ 要 | ❌ 不要 |
|------|--------|
| 短句，15 字以内为主 | 长句堆砌 |
| 口语化、接地气 | 文言腔、掉书袋 |
| 对话用「」括号 | " " 西式引号 |
| 少量方言（院坝、鸡叫头遍） | 过度方言影响理解 |
| 民俗细节一笔带过 | 给读者上课解释术语 |
| 情绪靠场景和动作传递 | "我感到一阵恐惧" |
| 结尾留白 | 把所有答案都说清楚 |

### 4.2 叙事视角

- **核心视角**：第一人称学徒（"我"），年轻、半吊子，跟读者一样懵
- **师父角色**：话少、沉稳、看穿不说穿，像武侠片里的扫地僧
- **配角**：村民、屠户、路人 — 每个都只出现一次，用完即走

### 4.3 语言样本对比

```
❌ AI味：
"夜色如墨，寂静的村庄笼罩在一片诡异的氛围之中。我心中涌起一股莫名的恐惧，
总觉得有什么看不见的东西正在黑暗中窥视着我……"

✅ 人写的（崔走召风格）：
"天黑透了。村子静得不对劲，连狗都不叫。我后脊梁一凉，脚步就慢了。
——不对。有人在看我。"
```

## 七、传递给 PM 的关键问题

PM 在写 PRD 前需要明确的：

1. **内容生产方式**？
   - A) 纯人工撰写（质量最高，产能有限）
   - B) AI 辅助 + 人工润色（产能较高，需确保不像 AI）
   - C) 训练专用模型自动生成（技术壁垒高，冷启动成本大）

2. **产品形态**？
   - A) 独立 App（需开发 iOS + Android）
   - B) 微信小程序（快速触达，审核风险）
   - C) 网页/PWA（开发成本最低，可添加到主屏幕）

3. **随机推荐策略**？
   - A) 完全随机（每次不一样）
   - B) 按题材随机（用户可选分类）
   - C) 智能推荐（根据阅读历史）

4. **商业化方向**？
   - A) 纯免费，个人项目
   - B) 内置广告
   - C) 订阅 / 按篇付费

5. **内容更新频率**？
   - 初始库存需要多少篇？建议至少 20-30 篇作为冷启动内容池
   - 后续更新节奏？建议每周 2-3 篇新故事
